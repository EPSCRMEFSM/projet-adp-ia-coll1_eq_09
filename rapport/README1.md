### Annexe B : Journal de traçabilité de l'ingénierie des prompts (Equipe COLL1_EQ_09)

Ce journal retrace l'évolution de l'outil d'aide à la décision pour le Lancer de Poids (DI-POI) développé dans le cadre de l'Atelier de Production Didactique (ADP 2026).

| Version | Date | Problème rencontré | Dialogue avec l'IA (Extrait de prompt) | Solution / Sortie obtenue |
| :--- | :--- | :--- | :--- | :--- |
| **V_1.0** | 14 mai | Manque la case des performances. | *« D'après mon code actuel j'ai trouvé que la case des performances est absente trouver le problème et résolu par l'ajoute des performances A cote des observables. »* | Ajout d'une case des performances. |
| **V_1.1** | 15 mai | Problème de lecture des données CSV de performances fictives. | *« Lors de mon utilisation de l'outil html j'ai trouvés que les donnes csv pas lecture, trouvés et résolu le problème »* | Outil fait la lecture donnes csv correctement. |
| **V_1.2** | 23 mai | L'outil manque d'un moyen de revenir rapidement à l'écran de départ sans recharger la page. | *« je veut ajouter le bouton de retour a l'accueille seulement sans modifier aucun chose dans le code. »* | Ajout d'un bouton "Accueil" dans le header déclenchant la fonction goStep(1). |
| **V_1.3** | 23 mai | Fatigue visuelle sur fond sombre / Risque de perte de données. | *« Ajouter bouton pour activer et désactiver mode sombre. (...) Je veut appliquer le sauvegarde locale automatique (...) »* | Mode light/dark dynamique + Sauvegarde locale automatique (localStorage) et Modal "Fiche élève" avec radar Canvas. |
| **V_1.4** | 23 mai | Impossible de suivre visuellement l'avancée du cycle et de comparer les élèves. | *« je veut ajouter Barre de progression du cycle Dans l'étape 4 [...] et. Comparaison avant/après »* | Timeline interactive à cocher (% du cycle) et Étape 5 de comparaison de deux fichiers CSV (calcul des deltas). |
| **V_1.5** | 23 mai | Manque de flexibilité pour imprimer uniquement la page courante. | *« tu peut ajouter un fonctionne pour la possibilité d'imprimer le produit de chaque page »* | Menu déroulant d'impression : Page courante, étape spécifique ou rapport complet. |
| **V_1.6** | 26 mai / 1 juin | Gestion multi-classes, mode tablette et prise en main difficile. | *« tu peut ajouter seulement Gestion de plusieurs classes [...] et Mode tablette terrain (...) Tu peux ajouter seulement. Tutoriel intégré »* | Onglets multi-classes, Mode Tablette (gros boutons tactiles A/B/C) et modal tutoriel interactif en 6 slides. |
| **V_1.7** | 12 juin | Besoin d'un support physique vierge pré-rempli avec les noms des élèves. | *« je Ve ajouter un bouton d'exportation de la grille de mon outil rempli par des donnes des noms »* | Création du bouton "Export Grille CSV" générant un fichier structuré avec colonnes d'observables vides. |
| **V_1.8** | 12 juin | Code fonctionnel mais non structuré pour un déploiement autonome. | *« Demande à un développeur de finaliser un outil EPS d'aide à la décision pour le rendre autonome, responsive, sécurisé localement et déployable sur GitHub Pages. »* | Adaptation globale du code : options d'import/export (JSON, CSV, PDF), design responsif et chemins relatifs pour GitHub Pages. |
